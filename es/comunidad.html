Proyectos y comunidades ya existentes

Si está contribuyendo o ampliando un proyecto existente, casi siempre es más fácil continuar usando ese proyecto licencia. Para encontrar su licencia, busque un archivo llamado LICECIAE o COPIA, y hojear el proyecto README. Si no podes encontrar una licencia, preguntale a los mantenedores.

Dependiendo de la licencia del proyecto original, usando la misma la licencia puede ser un requisito, no solo lo más fácil de hacer. (Ver la misma condición de licencia de alguna licencia.)

Algunas comunidades tienen fuertes preferencias por licencias particulares. Si usted quieren participar en uno de estos, será más fácil utilizar su licencia preferida, incluso si está comenzando un nuevo proyecto sin dependencias existentes. Algunos ejemplos incluyen:

  • Apache requiere Apache License 2.0
  • Cloud Native Computing Foundation requires Apache License 2.0
  • GNU requiere GNU GPLv3 para la mayoria de los programas
  • paquetes npm usa abrumadoramente el MIT o el muy similar ISC licencia
  • OpenBSD tiene preferencia por ISC License
  • Rust crates están abrumadoramente autorizadas bajo ambas MIT y Apache License 2.0
  • WordPress plugins y temas deben ser GNU GPLv2 (o posterior)

Las comunidades vienen en todas las formas y tamaños, y más de una comunidad podría ser pertinente (por ejemplo, tenga en cuenta su compania si trabajas para uno). Los ejemplos anteriores están muy bien establecido. Si la comunidad para la que está construyendo un proyecto no tiene tradiciones de concesión de licencias inamovibles, o no vea su proyecto como parte de una comunidad en particular, eso está bien: haga su propia elección de una licencia de código abierto.
